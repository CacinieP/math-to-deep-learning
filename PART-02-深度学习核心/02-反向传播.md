# 反向传播

> 反向传播就是**链式法则**。损失 $L$ 对参数 $\theta$ 的梯度 $\partial L/\partial \theta$，是从输出端往输入端、用局部雅可比矩阵相乘一层层传回去的过程。所有深度学习框架的 `loss.backward()` 干的都是这一件事。

**难度**：[标准]（需要一元/多元链式法则 + 矩阵运算 + 前向传播）

> 本文与 [[轴线C/01-梯度→反向传播]] 互补：那篇侧重链式法则的数学推导与 Jacobi 矩阵，本篇侧重**自动微分框架的实现视角**与**前向/反向模式的对比**。

## 一、纯数学版：链式法则的三种叙述

### 1.1 单变量链式法则

若 $L = L(y)$，$y = y(x)$：

$$\frac{dL}{dx} = \frac{dL}{dy} \cdot \frac{dy}{dx}$$

> 📖 链式法则：[[Mathematics-Universe/03-高等数学/02-一元微分学/一元微分学详解.md]]

### 1.2 多变量链式法则

若 $L \in \mathbb{R}$，$\mathbf{y} \in \mathbb{R}^m$，$\mathbf{x} \in \mathbb{R}^n$，且 $L = L(\mathbf{y}(\mathbf{x}))$：

$$\frac{\partial L}{\partial x_j} = \sum_{i=1}^{m} \frac{\partial L}{\partial y_i} \frac{\partial y_i}{\partial x_j}$$

写成矩阵：$\frac{\partial L}{\partial \mathbf{x}} = \frac{\partial L}{\partial \mathbf{y}} \cdot J_{\mathbf{y}\to\mathbf{x}}$，其中 $J$ 是雅可比矩阵。

> 📖 Jacobi 矩阵：[[Mathematics-Universe/03-高等数学/04-多元微分学/多元微分学详解.md]]

### 1.3 计算图视角

把函数拆成基本运算的**有向无环图（DAG）**，每个节点是一个中间变量：

```
x₁   x₂
 \   /
  \ /
   z = x₁·x₂        ← 节点
   |
   y = sin(z)
   |
   L = y²
```

**链式法则 = 沿 DAG 的路径把局部导数相乘、在分叉处相加。**

## 二、为什么是"反向"而不是"正向"

### 2.1 正向模式（forward-mode AD）

从输入出发，每个节点同时算**值**和**对该输入的导数**。

对一个输出、$n$ 个输入的函数：需要跑 $n$ 遍正向，才能拿到对所有输入的导数。**代价 $O(n)$ 倍。**

### 2.2 反向模式（reverse-mode AD / backprop）

从输出出发，每个节点算**值**（前向）和**输出对本节点的导数**（反向）。

对**一个标量输出**（损失）、$n$ 个参数的函数：只需跑 **1 遍前向 + 1 遍反向**，就拿到损失对所有参数的梯度。**代价 $O(1)$ 倍。**

### 2.3 为什么深度学习用反向

神经网络正是"**单标量损失 $L$，海量参数 $\theta$**"的结构——反向模式是渐近最优选择。

| 模式 | 输出数 | 输入数 | 代价 |
|------|--------|--------|------|
| 正向 AD | $m$ | $n$ | $O(m \cdot \text{前向代价})$ |
| 反向 AD | $m$ | $n$ | $O(n \cdot \text{前向代价})$ |

神经网络 $m=1, n\approx 10^6\sim10^{11}$，反向完胜。

## 三、反向传播的四步流程

### 3.1 前向：记录计算图

```
z₁ = W₁x + b₁      h₁ = σ(z₁)
z₂ = W₂h₁ + b₂     h₂ = σ(z₂)
ŷ  = W₃h₂ + b₃
L  = ℓ(ŷ, y)
```

每步把输入、输出、运算类型存起来，形成 DAG。

### 3.2 初始化：损失对自身的导数

$$\frac{\partial L}{\partial L} = 1$$

### 3.3 反向：逐节点传梯度

对每个节点，用**局部雅可比 × 上游传来的梯度**：

$$\bar{v} := \frac{\partial L}{\partial v}$$

- 损失端：$\bar{L} = 1$
- 经损失函数：$\bar{\hat{y}} = \frac{\partial L}{\partial \hat{y}}$
- 经仿射层 $z_l = W_l h_{l-1} + b_l$：

$$\bar{h}_{l-1} = W_l^\top \bar{z}_l, \qquad \bar{W}_l = \bar{z}_l h_{l-1}^\top, \qquad \bar{b}_l = \bar{z}_l$$

- 经激活 $h_l = \sigma(z_l)$（逐元素）：$\bar{z}_l = \bar{h}_l \odot \sigma'(z_l)$

### 3.4 累加到参数

参数的梯度 $\bar{W}_l, \bar{b}_l$ 就是我们要的 $\partial L/\partial W_l, \partial L/\partial b_l$，直接用于更新。

## 四、一个完整的手算例子

网络：$z = w x + b$，$y = \text{ReLU}(z)$，$L = (y - t)^2/2$。取 $x=2, w=1, b=0.5, t=1$。

**前向**：
- $z = 1\cdot 2 + 0.5 = 2.5$
- $y = \max(0, 2.5) = 2.5$
- $L = (2.5-1)^2/2 = 1.125$

**反向**：
- $\bar{L} = 1$
- $\bar{y} = y - t = 1.5$
- $\bar{z} = \bar{y} \cdot \sigma'(z) = 1.5 \cdot 1 = 1.5$（ReLU 在 $z>0$ 处导数为 1）
- $\bar{w} = \bar{z} \cdot x = 1.5 \cdot 2 = 3$
- $\bar{b} = \bar{z} \cdot 1 = 1.5$
- $\bar{x} = \bar{z} \cdot w = 1.5 \cdot 1 = 1.5$

所以 $\partial L/\partial w = 3$，$\partial L/\partial b = 1.5$。

## 五、工程实现层：PyTorch autograd

### 5.1 一个例子验证手算

```python
import torch

x = torch.tensor(2.0)
w = torch.tensor(1.0, requires_grad=True)
b = torch.tensor(0.5, requires_grad=True)
t = torch.tensor(1.0)

z = w * x + b
y = torch.relu(z)
L = 0.5 * (y - t) ** 2

L.backward()   # 触发反向传播
print(w.grad, b.grad)   # tensor(3.) tensor(1.5)  ← 与手算一致
```

### 5.2 自动微分的核心 API

```python
# 每个 tensor 有 .requires_grad 标志, 决定是否追踪
# 每个运算的 backward 定义在 autograd.Function 里
# backward() 做两件事:
#   1. 拓扑排序计算图
#   2. 从叶子端反向, 逐节点调用 .backward 应用链式法则
```

### 5.3 批量反向传播（矩阵形式）

对批量数据 $X \in \mathbb{R}^{B \times d}$，整个批次的梯度是样本梯度的平均：

```python
linear = torch.nn.Linear(784, 10)
X = torch.randn(32, 784)         # batch=32
logits = linear(X)               # (32, 10)
loss = F.cross_entropy(logits, torch.randint(0, 10, (32,)))
loss.backward()
print(linear.weight.grad.shape)  # torch.Size([10, 784])
```

## 六、三个工程陷阱

### 6.1 梯度清零

PyTorch 默认**累加**梯度，每个 batch 前必须 `optimizer.zero_grad()`，否则梯度会跨 batch 累计。

### 6.2 反向后计算图被释放

`backward()` 默认释放计算图（`retain_graph=False`）。第二次 `backward()` 会报错，除非显式保留。这避免显存泄漏。

### 6.3 detach 切断梯度

`tensor.detach()` 把一个 tensor 从计算图摘下，反向传播到此为止。用于：冻结参数、记录中间值、对抗生成网络中判别器不回传到生成器。

## 八、直觉总结

```
前向:  x → z₁ → h₁ → z₂ → h₂ → ŷ → L
        每步算"值"

反向:  L → ŷ → h₂ → z₂ → h₁ → z₁ → x
        每步问"我对前面有多大影响?"
        答案 = 局部影响 × 上游影响  (链式法则)

参数梯度 Ḃ_l, b̄_l 直接喂给优化器去更新参数
```

**反向传播 = 链式法则 + 计算图 + 拓扑排序。** 不是新数学，是古老定理的高效实现。
