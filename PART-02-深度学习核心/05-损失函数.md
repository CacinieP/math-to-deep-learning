# 损失函数

> 损失函数 $\ell(\hat y, y)$ 衡量"预测错得有多离谱"。每种损失背后都对应一个**概率假设**——最小化损失，等价于在某个概率模型下做**极大似然估计（MLE）**。MSE 对应高斯噪声，交叉熵对应分类的负对数似然，KL 散度则衡量两个分布的差距。

**难度**：[标准]（需要概率分布 + 极大似然 + 一点信息论）

## 一、纯数学版：损失 = 负对数似然

### 1.1 极大似然与损失的关系

设数据来自参数模型 $p(y|x;\theta)$。MLE 找让数据概率最大的 $\theta$：

$$\theta^* = \arg\max_\theta \prod_i p(y_i|x_i;\theta) = \arg\max_\theta \sum_i \log p(y_i|x_i;\theta)$$

取负、除以 $N$，就得到**经验损失**：

$$\mathcal{L}(\theta) = -\frac{1}{N}\sum_i \log p(y_i|x_i;\theta)$$

**关键结论**：**任何损失函数都对应一个负对数似然（NLL）**。选损失 = 选概率假设。

> 📖 极大似然：[[Mathematics-Universe/05-概率论与数理统计/06-数理统计基础/数理统计基础详解.md]]

### 1.2 MSE ⇔ 高斯噪声假设

假设 $y = f_\theta(x) + \varepsilon$，$\varepsilon \sim \mathcal{N}(0, \sigma^2)$。则：

$$\log p(y|x;\theta) = -\frac{(y - f_\theta(x))^2}{2\sigma^2} - \frac{1}{2}\log(2\pi\sigma^2)$$

$$\Rightarrow \mathcal{L}_{\text{MSE}} = \frac{1}{N}\sum_i (y_i - f_\theta(x_i))^2$$

**MSE 的本质**：假设噪声是高斯的。用它做分类（噪声非高斯）就是误用。

### 1.3 交叉熵 ⇔ 分类似然

分类任务中 $y \in \{1,\ldots,C\}$，模型输出类别概率 $\hat p_c = P(y=c|x)$。似然：

$$\log p(y|x) = \log \hat p_y$$

$$\Rightarrow \mathcal{L}_{\text{CE}} = -\log \hat p_y = -\sum_c \mathbf{1}[y=c]\log\hat p_c$$

这正是**交叉熵损失**。用 softmax 输出 $\hat p$，得：

$$\hat p_c = \frac{e^{z_c}}{\sum_k e^{z_k}} \Rightarrow \mathcal{L}_{\text{CE+softmax}} = -z_y + \log\sum_k e^{z_k}$$

> 📖 熵与交叉熵：[[Mathematics-Universe/05-概率论与数理统计/04-数字特征/数字特征详解.md]]

### 1.4 KL 散度：分布间的"距离"

$$D_{\text{KL}}(p \| q) = \sum_c p_c \log\frac{p_c}{q_c} = \underbrace{-\sum_c p_c\log q_c}_{H(p,q)\text{ 交叉熵}} + \underbrace{\sum_c p_c\log p_c}_{-H(p)}$$

**性质**：$D_{\text{KL}} \geq 0$，等号当 $p=q$。**不是对称的**，故严格说不是距离。

**与交叉熵的关系**：最小化交叉熵 $H(p,q)$ 等价于最小化 $D_{\text{KL}}(p\|q)$（因 $H(p)$ 对 $q$ 是常数）。这就是为什么分类用 CE。

## 二、应用映射层：常见损失的直觉与适用

### 2.1 回归损失

| 损失 | 公式 | 噪声假设 | 特点 |
|------|------|---------|------|
| MSE | $(y-\hat y)^2$ | 高斯 | 对离群点敏感（平方放大） |
| MAE | $\|y-\hat y\|$ | 拉普拉斯 | 稳健，但零点不可导 |
| Huber | 平方/线性分段 | 混合 | 兼顾稳定与可导 |
| 分位数 | $\rho_\tau(y-\hat y)$ | 非对称拉普拉斯 | 给出预测区间 |

```
   损失
    │
 MSE╱ ╲      平方增长, 离群点惩罚重
    │   ╲
MAE │    ╲___  线性增长, 稳健
    └──────────── 预测误差
```

### 2.2 分类损失

| 损失 | 公式 | 用途 |
|------|------|------|
| 二元 CE | $-[y\log\hat p + (1-y)\log(1-\hat p)]$ | 二分类 |
| 多类 CE | $-\log\hat p_y$ | 多分类（配合 softmax） |
| Focal | $-(1-\hat p_y)^\gamma \log\hat p_y$ | 类别不平衡，难样本聚焦 |
| Hinge | $\max(0, 1 - y\hat y)$ | SVM，最大间隔 |

### 2.3 Focal Loss：为什么普通 CE 在不平衡数据上失效

类别极度不平衡（如检测中背景占 90%），普通 CE 被多数类淹没。Focal 加权因子 $(1-\hat p_y)^\gamma$ **降低易分样本的权重**：

- $\hat p_y$ 高（已经很对）：$(1-\hat p_y)^\gamma$ 接近 0，几乎不贡献损失
- $\hat p_y$ 低（还错着）：贡献几乎不变

效果：优化注意力集中在**难样本**上。

## 三、为什么分类用 CE 不用 MSE

### 3.1 一个数值实验的教训

用 MSE + sigmoid 输出做分类时，梯度在错得离谱处会**消失**：

$$\frac{\partial L_{\text{MSE}}}{\partial z} = 2(\hat y - y)\cdot\sigma'(z)$$

当 $\hat y$ 接近 0 或 1（饱和），$\sigma'(z) \to 0$，梯度消失——**错得越狠学得越慢**。

### 3.2 CE 的优美性质

CE + softmax 的梯度：

$$\frac{\partial L_{\text{CE}}}{\partial z_c} = \hat p_c - \mathbf{1}[y=c]$$

**线性于预测误差**：错得越狠梯度越大，学习越快。这是分类任务几乎全用 CE 的根本原因。

## 四、工程实现层：PyTorch 损失函数

### 4.1 标准用法

```python
import torch.nn as nn
import torch.nn.functional as F

# 回归
mse = nn.MSELoss()
loss = mse(predicted, target)         # predicted, target 同形

# 多分类(配合 logits, 内含 softmax, 数值稳定)
loss = F.cross_entropy(logits, labels)  # logits: (B, C), labels: (B,)

# 二分类
loss = F.binary_cross_entropy_with_logits(logits, targets)
```

### 4.2 自定义损失：带类权重的 CE

```python
def weighted_ce(logits, labels, class_weights):
    # class_weights: (C,), 处理类别不平衡
    log_probs = F.log_softmax(logits, dim=1)
    loss = -class_weights[labels] * log_probs[range(len(labels)), labels]
    return loss.mean()
```

### 4.3 Focal Loss 实现

```python
def focal_loss(logits, labels, gamma=2.0):
    log_probs = F.log_softmax(logits, dim=1)
    probs = log_probs.exp()
    focal_weight = (1 - probs[range(len(labels)), labels]) ** gamma
    return -(focal_weight * log_probs[range(len(labels)), labels]).mean()
```

### 4.4 KL 散度（VAE 等用）

```python
# 标准 VAE 的 KL 项: D_KL(N(μ,σ) || N(0,1))
kl = -0.5 * torch.sum(1 + logvar - mu.pow(2) - logvar.exp(), dim=1).mean()
```

## 五、损失的数值稳定性

### 5.1 LogSumExp 技巧

直接算 $\log\sum e^{z_k}$ 会溢出。改用：

$$\log\sum_k e^{z_k} = m + \log\sum_k e^{z_k - m}, \quad m = \max_k z_k$$

PyTorch 的 `cross_entropy` 内部已用此技巧，**永远传 logits 而非 probs**。

### 5.2 label smoothing

把硬标签 $y=c$ 换成软分布 $y_{\text{smooth}} = (1-\alpha)\mathbf{1}[y=c] + \alpha/C$。缓解过自信，提升校准。

## 七、直觉总结

```
损失函数 ≠ 随便选, 每个都对应一个"世界的假设":

  MSE      = 假设噪声是高斯        → 回归
  交叉熵    = 假设目标是类别        → 分类
  Huber    = 假设有离群点          → 稳健回归
  Focal    = 假设类别不平衡        → 难样本聚焦
  KL散度   = 假设目标是分布        → VAE/Diffusion

选错损失 = 用错概率模型, 再好的网络也学不好
```

**损失函数 = 你对"世界长什么样"的数学假设。** 数学是 MLE，工程是数值稳定的实现。
