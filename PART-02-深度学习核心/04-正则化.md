# 正则化

> 正则化是在优化目标里**主动加一个"惩罚项"**，让模型宁愿简单也不要过拟合：$\tilde L = L + \lambda \Omega(\theta)$。数学上，这是在**拟合能力**与**复杂度**之间做权衡——本质是 Hilbert 空间里的**Tikhonov 正则化**或 Sobolev 空间里的**范数约束**。

**难度**：[标准]（需要偏差-方差分解 + 范数 + 一点泛函直觉）

## 一、纯数学版：为什么要惩罚复杂度

### 1.1 过拟合 = 经验风险最小化的病态

给定数据 $(x_i, y_i)_{i=1}^N$，朴素的训练目标是**经验风险最小化**：

$$\hat\theta = \arg\min_\theta \frac{1}{N}\sum_{i=1}^N \ell(f_\theta(x_i), y_i)$$

**问题**：当模型容量够大（参数够多），总能找到 $\theta$ 让训练损失为 0——但这样的 $\theta$ 几乎只记住了噪声，泛化崩塌。

### 1.2 正则化：结构风险最小化

加一个**复杂度惩罚** $\Omega(\theta)$：

$$\hat\theta = \arg\min_\theta \underbrace{\frac{1}{N}\sum \ell(f_\theta(x_i), y_i)}_{\text{拟合}} + \lambda \underbrace{\Omega(\theta)}_{\text{复杂度}}$$

$\lambda$ 控制"多在乎简单"。这是 Vapnik 的**结构风险最小化**原则。

### 1.3 偏差-方差分解的视角

对平方损失，期望泛化误差可分解为：

$$\mathbb{E}[(y - \hat y)^2] = \underbrace{(\mathbb{E}\hat y - y)^2}_{\text{偏差}^2} + \underbrace{\mathbb{E}[(\hat y - \mathbb{E}\hat y)^2]}_{\text{方差}} + \sigma^2$$

- 过拟合 = 低偏差、高方差
- 欠拟合 = 高偏差、低方差
- 正则化 = 牺牲一点偏差，换方差大幅下降

> 📖 偏差-方差：[[Mathematics-Universe/05-概率论与数理统计/06-数理统计基础/数理统计基础详解.md]]

### 1.4 L2 正则 = Tikhonov = 高斯先验

取 $\Omega(\theta) = \|\theta\|_2^2$，目标变成：

$$\tilde L = L(\theta) + \lambda \|\theta\|_2^2$$

**三种等价视角**：
1. **优化**：惩罚大权重，让解"光滑"
2. **约束**：等价于 $\|\theta\|_2 \leq R$ 的约束优化（Lagrange 对偶）
3. **贝叶斯**：等价于参数的高斯先验 $\theta \sim \mathcal{N}(0, \sigma^2/\lambda)$（MAP 估计）

### 1.5 L1 正则 = 稀疏 = 拉普拉斯先验

取 $\Omega(\theta) = \|\theta\|_1 = \sum_j |\theta_j|$：

$$\tilde L = L(\theta) + \lambda \|\theta\|_1$$

**几何**：L1 球有尖角，与等高线相切处常落在坐标轴上——使很多 $\theta_j = 0$，产生**稀疏解**。

**贝叶斯视角**：对应拉普拉斯先验 $\theta_j \sim \text{Laplace}(0, 1/\lambda)$。

## 二、应用映射层：从数学到深度学习的各种正则

### 2.1 权重衰减（Weight Decay）

L2 正则在神经网络里的名字。梯度变成：

$$\nabla \tilde L = \nabla L + 2\lambda \theta$$

更新时多了一项 $-2\eta\lambda\theta$——每次把权重**往原点拉**一点。

### 2.2 Dropout（2012）

训练时随机**置零**一部分神经元（概率 $p$）：

$$h_{\text{drop}} = \frac{m}{1-p} \odot h, \quad m_j \sim \text{Bernoulli}(1-p)$$

**数学解释**：
- **集成视角**：训练了指数多个子网络的隐式集成，测试时取平均
- **贝叶斯视角**（Gal & Ghahramani, 2016）：Dropout 等价于贝叶斯神经网络近似推断，能得到**不确定性估计**
- **正则视角**：禁止神经元过度依赖个别特征，强迫冗余表示

### 2.3 早停（Early Stopping）

验证集损失上升就停训。**等价于 L2 正则**：训练时间 $t$ 与权重衰减系数 $\lambda$ 有一一对应（梯度下降轨迹被截断 = 在原点附近的解）。

### 2.4 Batch Normalization（附带正则效果）

原本为解决内部协变量偏移，实践发现其随机性（mini-batch 统计）有轻微正则作用。

### 2.5 数据增强

对输入做随机变换（翻转、裁剪、加噪）。**本质**：扩充数据分布的支持集，降低过拟合。数学上等价于对输入加先验。

## 三、L1 vs L2 的几何直觉

### 3.1 等高线与约束区域的相切

```
      L2 (圆滑)              L1 (菱形, 有尖角)
       ┌─────┐                  ◇
      /  •   \                ╱ │ ╲     等高线与尖角
     |  •切点 |               │ •┊•│    相切于轴上 ⇒ θ_j=0
      \       /                ╲ │ ╱     ⇒ 稀疏解
       └─────┘                  ◇
    (切点通常不在轴上)        (切点常落在尖角=坐标轴)
```

**结论**：L1 产生稀疏（特征选择），L2 产生小而分散的权重。

### 3.2 Elastic Net = L1 + L2

$$\Omega(\theta) = \alpha \|\theta\|_1 + (1-\alpha)\|\theta\|_2^2$$

兼顾稀疏性与稳定性，常见于高维稀疏数据。

## 四、工程实现层：PyTorch 正则化

### 4.1 权重衰减（最简单）

```python
# 直接在 optimizer 里指定 weight_decay
opt = torch.optim.Adam(model.parameters(), lr=1e-3, weight_decay=1e-4)
# weight_decay 就是 L2 正则的 λ; PyTorch 自动加 2λ·θ 到梯度
```

### 4.2 Dropout

```python
import torch.nn as nn

class Net(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(784, 256)
        self.drop = nn.Dropout(p=0.5)   # 训练时 50% 置零, 测试时自动关闭
        self.fc2 = nn.Linear(256, 10)
    def forward(self, x):
        x = self.drop(torch.relu(self.fc1(x)))
        return self.fc2(x)
# model.train() 启用 dropout; model.eval() 关闭(自动 rescale)
```

### 4.3 早停

```python
best_val = float('inf')
patience, bad = 5, 0
for epoch in range(100):
    train_loss = train_one_epoch()
    val_loss = validate()
    if val_loss < best_val:
        best_val = val_loss; bad = 0
        torch.save(model.state_dict(), 'best.pt')
    else:
        bad += 1
        if bad >= patience:
            print("Early stop at epoch", epoch); break
model.load_state_dict(torch.load('best.pt'))
```

### 4.4 L1 正则（需手动加）

```python
l1_lambda = 1e-5
loss = loss_fn(logits, y)
l1 = sum(p.abs().sum() for p in model.parameters())
loss = loss + l1_lambda * l1   # 手动加 L1 惩罚
loss.backward()
```

## 五、正则化强度的选择

$\lambda$ 是超参数，靠验证集调。一般规律：

| $\lambda$ | 现象 | 适用 |
|-----------|------|------|
| 太大 | 欠拟合，训练损失也高 | — |
| 太小 | 过拟合，训练低/验证高 | — |
| 合适 | 训练略升，验证最低 | 最佳 |

**经验**：小数据集需要强正则（大 $\lambda$ + dropout）；大数据集正则可减弱。

## 七、直觉总结

```
过拟合: 模型把训练集的噪声也背了下来, 泛化差
正则化: 在损失里加一句"别太复杂"

  L2 (权重衰减): 大权重受罚 ⇒ 所有权重都小一点 (光滑)
  L1:           绝对值受罚 ⇒ 很多权重直接归零 (稀疏, 特征选择)
  Dropout:      随机删神经元 ⇒ 强迫冗余, 隐式集成
  早停:         别训太久 ⇒ 等价于 L2

数学根: 偏差-方差权衡; 泛函: 函数空间范数约束
```

**正则化 = 给"只顾拟合"的优化器装上刹车。** 数学是范数惩罚，工程是各种随机性。
