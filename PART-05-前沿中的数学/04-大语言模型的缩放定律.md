# 大语言模型的缩放定律

> 大模型的能力随参数量 $N$、数据量 $D$、算力 $C$ 的增长遵循**平滑的幂律**：损失 $L = L_\infty + (N/N_0)^{-\alpha}$。这不是经验拟合——幂律背后是**统计学习、临界现象、相变**的物理直觉。理解缩放定律，才能理性预测"再投 10 倍算力能换多少能力"。

**难度**：[前沿]（需要幂律 + 对数 + 统计直觉）

## 一、纯数学：幂律与标度

### 1.1 幂律

$$y = a x^b \quad \Leftrightarrow \quad \log y = \log a + b \log x$$

**双对数图上是直线**。幂律在自然现象里 ubiquitous：地震（Gutenberg-Richter）、城市人口、单词频率（Zipf）。

### 1.2 神经网络的缩放定律（Kaplan 2020）

OpenAI 发现，自回归语言模型的测试损失随三个量各自呈幂律：

$$L(N) = \left(\frac{N_0}{N}\right)^{\alpha_N}, \quad L(D) = \left(\frac{D_0}{D}\right)^{\alpha_D}, \quad L(C) = \left(\frac{C_0}{C}\right)^{\alpha_C}$$

典型指数 $\alpha \approx 0.05 \sim 0.1$（缓慢但持续下降）。

**意义**：损失是**可预测的光滑函数**——投 10 倍算力，损失下降固定比例。

## 二、Chinchilla：算力的最优分配

### 2.1 Kaplan 的失误

Kaplan 2020 的建议：固定数据，扩参数。导致许多大模型**参数过剩、数据不足**。

### 2.2 Chinchilla 的修正（Hoffmann 2022）

DeepMind 发现：固定算力 $C$ 时，最优的参数 $N^*$ 和数据 $D^*$ 应**等比例增长**：

$$N^* \propto C^{0.5}, \quad D^* \propto C^{0.5}$$

即 **20:1 的 token 数与参数比**。Chinchilla 用 700 亿参数 + 1.4 万亿 token，超过了 1750 亿参数的 GPT-3。

### 2.3 损失的联合公式

$$L(N, D) = E + \frac{A}{N^\alpha} + \frac{B}{D^\beta}$$

- $E$：不可约损失（数据的熵下界）
- $A/N^\alpha$：参数不足损失
- $B/D^\beta$：数据不足损失

**最优分配** = 让两项平衡，而不是让某一项主导。

## 三、幂律的理论解释

### 3.1 临界现象类比（统计物理）

物理相变附近，序参量随温度呈幂律：$M \sim |T - T_c|^\beta$。**临界点处关联长度发散**，系统对扰动有幂律响应。

**类比猜想**：大模型接近某种"临界态"，loss 对算力呈幂律。这是为什么缩放律"光滑可预测"。

### 3.2 信息论视角

损失的不可约下界 $E$ 是真实数据的**熵率**：

$$E = H(\text{下一词} | \text{上下文})$$

无论模型多大，都至少要这么多比特预测下一词。幂律描述的是**接近这个下界的速度**。

> 📖 熵：[[轴线E/01-熵→交叉熵损失]]

### 3.3 偏差-方差分解

参数不足 → 偏差大；数据不足 → 方差大。最优分配在偏差和方差间平衡——这是经典统计学习的现代回响。

## 四、涌现能力：相变的迹象

### 4.1 涌现现象

某些能力（少样本推理、数学解题）在小模型里几乎没有，参数越过某个阈值后**突然出现**——称为**涌现**。

### 4.2 相变解释

涌现酷似物理**相变**：水结冰、磁化等，在临界点性质突变。

**数学描述**：能力 $A$ 与规模 $N$ 的关系可能是**阶跃函数**而非幂律：

$$A(N) \approx \mathbf{1}[N > N_c]$$

$N_c$ 是临界规模。幂律描述损失（连续量），涌现描述能力（离散能力）。

### 4.3 争议

有研究（Schaeffer 2023）质疑：涌现可能只是**评估指标的非线性**（如用 exact-match 而非部分分数）。换指标后幂律重现。

## 五、工程实践：用缩放律做规划

### 5.1 拟合缩放律

```python
import numpy as np
from scipy.optimize import curve_fit

# 训练多个小模型, 记录 (N, D, loss)
def scaling_law(X, A, alpha, B, beta, E):
    N, D = X
    return E + A / N**alpha + B / D**beta

# 拟合
popt, _ = curve_fit(scaling_law, (Ns, Ds), losses)
# 用拟合结果外推: 投 10x 算力, 损失会到多少?
```

### 5.2 计算 Chinchilla 最优点

```python
# 给定算力 C, 求最优 N, D
# N* ≈ C^0.5, D* ≈ 20 * N*
def optimal_allocation(C):
    N = (C / 20) ** 0.5  # 简化
    D = 20 * N
    return N, D
```

### 5.3 实践教训

- **数据常被低估**：Chinchilla 之前，大家堆参数不堆数据
- **幂律不会无限持续**：接近熵下界 $E$ 时饱和
- **质量 > 数量**：过滤后的高质量数据，等效算力翻倍

## 七、直觉总结

```
缩放定律: 损失随 N, D, C 平滑幂律下降 (双对数直线)
Chinchilla: 固定算力下, 参数和数据等比增长最优 (20 token/参数)
下界 E: 数据的真实熵, 不可超越
涌现: 某些能力在临界规模"突然出现"(相变迹象, 有争议)
```

**缩放律让大模型的能力变得可预测。** 数学是幂律，工程是算力分配的最优化。
