# 数值稳定性

> 理论上 $\log\sum e^{x_i}$ 是良定义的，GPU 上算出来可能是 `inf` 或 `nan`。浮点数只有有限精度——32 位浮点能表示的范围是 $10^{\pm 38}$。**数值稳定性**研究：怎样排布运算顺序，让数学正确的公式在有限精度下不出错。

**难度**：[标准]（需要浮点数基础 + 对数）

## 一、浮点数：有限精度的世界

### 1.1 IEEE 754 浮点

| 类型 | 位数 | 最小正规数 | 最大 | 精度（有效位） |
|------|------|-----------|------|--------------|
| float32 | 32 | $\approx 1.2\times 10^{-38}$ | $\approx 3.4\times 10^{38}$ | ~7 位十进制 |
| float16 | 16 | $\approx 6.1\times 10^{-5}$ | 65504 | ~3 位 |
| float64 | 64 | $\approx 2.2\times 10^{-308}$ | $\approx 1.8\times 10^{308}$ | ~16 位 |

**关键限制**：
- 上溢（overflow）：超出最大值 → `inf`
- 下溢（underflow）：小于最小正规数 → 0（丢失梯度）

### 1.2 误差的累积

浮点运算**不满足结合律**：$(a+b)+c \neq a+(b+c)$。大数加小数会丢精度：

```python
1e20 + 1 - 1e20  # 理论=1, float64 可能=0
1e9 + 1e-9       # float32 下 1e-9 被吞掉
```

**深度学习影响**：百万次累加的梯度、softmax 的指数、BN 的方差——都受此影响。

## 二、经典病态：softmax 与 LogSumExp

### 2.1 原始 softmax 的爆炸

$$\text{softmax}(\mathbf{z})_i = \frac{e^{z_i}}{\sum_k e^{z_k}}$$

当 $z_i = 1000$，$e^{1000} \gg 10^{38}$ → `inf`，分母 `inf`，结果 `nan`。

### 2.2 LogSumExp 技巧

**关键恒等式**：

$$\log\sum_k e^{z_k} = m + \log\sum_k e^{z_k - m}, \quad m = \max_k z_k$$

减去最大值后，最大的指数项变成 $e^0 = 1$，其余都 $<1$，不会溢出。

### 2.3 稳定的 softmax

$$\text{softmax}(\mathbf{z})_i = \frac{e^{z_i - m}}{\sum_k e^{z_k - m}}$$

分子分母同减 $e^m$，数学等价但数值稳定。

```python
import torch
import torch.nn.functional as F

# ❌ 错误: 手算易溢出
z = torch.tensor([1000., 1001., 1002.])
# torch.exp(z) → [inf, inf, inf]

# ✅ 正确: 传 logits, 内部用 LogSumExp
loss = F.cross_entropy(z.unsqueeze(0), torch.tensor([0]))  # 稳定
```

## 三、对数域计算：乘法变加法

### 3.1 为什么用对数

概率连乘 $\prod_i p_i$ 当 $p_i$ 小时迅速下溢到 0。取对数变连加：

$$\log\prod_i p_i = \sum_i \log p_i$$

**应用**：
- 极大似然：用 NLL（负对数似然）而非纯似然
- 贝叶斯：后验 ∝ 似然 × 先验，对数域相加
- 序列模型：对数概率累加

### 3.2 对数域的减法（logsumexp 的逆）

要算 $\log(e^a - e^b)$（$a > b$）：

$$\log(e^a - e^b) = a + \log(1 - e^{b-a})$$

避免直接算 $e^a - e^b$ 的抵消误差。

## 四、梯度爆炸与消失

### 4.1 浮点视角

- 梯度 $> 10^{38}$ → `inf`（爆炸），更新后权重变 `nan`
- 梯度 $< 10^{-38}$ → `0`（消失），参数不更新

**根本原因**：深网络梯度是各层雅可比的连乘，雅可比特征值稍偏离 1 就会指数放大/缩小。

### 4.2 缓解

- **梯度裁剪**：$\|\nabla\| > \tau$ 时缩放回 $\tau$
- **残差连接**：$I$ 项保梯度不消失
- **LayerNorm**：归一化每层激活，控制尺度

## 五、工程实现层

### 5.1 梯度裁剪

```python
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
# 梯度 L2 范数超过 1.0 就等比缩放
```

### 5.2 用 float64 调试

```python
# 怀疑数值问题时, 临时切到 float64 验证
model = model.double()
# 若 float64 正常而 float32 异常, 就是数值精度问题
```

### 5.3 检测 nan/inf

```python
torch.isnan(tensor).any()   # 有无 nan
torch.isinf(tensor).any()   # 有无 inf
# 训练循环里定期检查, 早发现
```

### 5.4 混合精度训练

```python
from torch.cuda.amp import autocast, GradScaler
scaler = GradScaler()
with autocast():           # 前向用 float16 加速
    loss = model(x)
scaler.scale(loss).backward()
scaler.step(optimizer)     # 梯度反缩放到 float32 更新
scaler.update()
```

## 七、直觉总结

```
浮点: 有限精度, 有上溢(inf)和下溢(0)
softmax: e^1000 = inf ⇒ 用 LogSumExp 减最大值
概率: 连乘下溢 ⇒ 取对数变连加
梯度: 连乘放大/缩小 ⇒ 裁剪 + 残差 + Norm
```

**数值稳定 = 让数学公式在有限精度下也算对。** 工程的核心技巧是"减最大值"和"取对数"。
