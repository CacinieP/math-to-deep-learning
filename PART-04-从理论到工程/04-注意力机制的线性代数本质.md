# 注意力机制的线性代数本质

> 剥开所有包装，注意力就是**三个矩阵的乘法**：$Q K^\top$ 算相似度（内积），softmax 归一化（行随机矩阵），再乘 $V$（加权平均）。整个过程是一组**行随机矩阵 × 值矩阵**——线性代数里最朴素的操作，赋予了它"软查询"的能力。

**难度**：[进阶]（需要矩阵乘法 + softmax + 内积空间）

## 一、纯数学：把注意力拆成矩阵运算

### 1.1 标准公式

$$\text{Attn}(Q, K, V) = \text{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}\right)V$$

逐项拆解：
- $Q \in \mathbb{R}^{N \times d_k}$：查询矩阵
- $K \in \mathbb{R}^{N \times d_k}$：键矩阵
- $V \in \mathbb{R}^{N \times d_v}$：值矩阵
- $QK^\top \in \mathbb{R}^{N \times N}$：相似度矩阵
- $\text{softmax}(\cdot)$ 逐行归一化 → $A \in \mathbb{R}^{N \times N}$（行随机）
- $AV \in \mathbb{R}^{N \times d_v}$：输出

> 📖 内积与相似度：[[Mathematics-Universe/04-线性代数/03-向量与向量空间/向量与向量空间详解.md]]

### 1.2 第 $i$ 行的语义

输出第 $i$ 行：$\text{out}_i = \sum_j A_{ij} V_{j:}$。

- $A_{ij}$ = "查询 $i$ 对键 $j$ 的关注度"
- $\text{out}_i$ = 所有值向量按关注度加权平均

**第 $i$ 个输出 = 用第 $i$ 个查询去"软检索"所有键值对**。

---

## 二、三个矩阵的角色

### 2.1 QK^⊤：相似度矩阵

$(QK^\top)_{ij} = \mathbf{q}_i \cdot \mathbf{k}_j$ —— 查询 $i$ 与键 $j$ 的内积。

**内积 = 相似度**（在归一化空间里）。这就是为什么用 $QK^\top$——把"语义相关性"转化为"向量内积"。

```
        键: k₁   k₂   k₃   k₄
查询 q₁:  [0.1, 0.9, 0.2, 0.05]   ← q₁ 与 k₂ 最相关
查询 q₂:  [0.7, 0.1, 0.1, 0.1 ]   ← q₂ 与 k₁ 最相关
```

### 2.2 softmax：行随机矩阵

softmax 把每行归一化成概率分布（和为 1）。结果矩阵 $A$ 是**行随机矩阵**：

$$\sum_j A_{ij} = 1, \quad A_{ij} \geq 0$$

**性质**：$A$ 作用在任何矩阵上，输出是行的**凸组合**（加权平均，不超出原值范围）。

### 2.3 V：值矩阵

$V$ 是被聚合的内容。$AV$ = 按注意力权重重新混合所有值向量。

### 2.4 整体 = 软查询表

传统查表：硬选一行。注意力：**软选所有行的凸组合**，权重由相似度决定。

---

## 三、softmax = 概率归一化

### 3.1 从内积到概率

$QK^\top/\sqrt{d_k}$ 是任意实数。softmax 转成概率：

$$A_{ij} = \frac{e^{(QK^\top)_{ij}/\sqrt{d_k}}}{\sum_k e^{(QK^\top)_{ik}/\sqrt{d_k}}}$$

**温度 $\sqrt{d_k}$** 控制分布尖锐程度。除以 $\sqrt{d_k}$ 是方差归一化（防饱和）。

> 📖 $\sqrt{d_k}$ 的方差分析：[[轴线D/02-Fourier分析→注意力机制]]

### 3.2 极限行为

- 温度 → 0：softmax → argmax（硬选最大）
- 温度 → ∞：softmax → 均匀（平均所有）

**softmax 的作用**：把连续相似度转成"软选择"——既关注最相关的，也不完全忽略其余。

---

## 四、多头 = 多组并行线性变换

### 4.1 多头的矩阵结构

把 $d$ 维拆成 $h$ 个 $d/h$ 子空间，每组独立做 $Q_i K_i^\top V_i$：

$$\text{head}_i = \text{Attn}(Q W_Q^i, K W_K^i, V W_V^i)$$
$$\text{MultiHead} = \text{Concat}(\text{head}_1, \ldots, \text{head}_h) W_O$$

**线性代数视角**：每个头在**不同的子空间投影**里做相似度匹配，捕获不同类型的关系。

### 4.2 与多视图分析类比

- 短时 Fourier 变换：多个窗口看信号
- 多头注意力：多个子空间看序列

## 五、工程实现层

### 5.1 一行注意力

```python
import torch, torch.nn.functional as F, math

def attention(Q, K, V):
    d_k = Q.size(-1)
    scores = Q @ K.transpose(-2, -1) / math.sqrt(d_k)  # (N, N) 相似度
    A = F.softmax(scores, dim=-1)                       # 行随机矩阵
    return A @ V                                        # (N, d_v)

Q = torch.randn(10, 64); K = torch.randn(10, 64); V = torch.randn(10, 64)
out = attention(Q, K, V)  # (10, 64)
```

### 5.2 可视化注意力矩阵

```python
import matplotlib.pyplot as plt
A = F.softmax(Q @ K.T / math.sqrt(64), dim=-1)
plt.imshow(A.detach().numpy(), cmap='viridis')
# 对角线常亮(自关注强), 某些行/列有强关注模式
```

### 5.3 因果遮罩（自回归）

```python
# 下三角掩码: 只看前面的 token
mask = torch.tril(torch.ones(N, N))
scores = scores.masked_fill(mask == 0, float('-inf'))
A = F.softmax(scores, dim=-1)  # 被掩位置概率为 0
```

## 六、线性代数视角能解释什么

### 6.1 注意力是低秩的吗

研究表明，长序列的注意力矩阵 $A$ 往往**近似低秩**——大量信息集中在少数主方向。这是 Linear Attention、Flash Attention 等加速方法的理论根据。

> 📖 谱分析：[[05-Transformer的谱分析]]

### 6.2 注意力 = 软聚类

$A$ 的行是软分配，$AV$ 是按分配加权均值。这等价于**软 K-means**——每个查询软分配到"簇"（值向量）。

### 6.3 为什么 Transformer 强

注意力是**全连接的**（任意两 token 直接交互，不像 RNN 要逐层传），又**数据驱动**（权重从输入学），还**可并行**（矩阵乘法）——三个线性代数优势叠加。

## 八、直觉总结

```
注意力 = 三个矩阵乘法 + 一个 softmax

QKᵀ:  查询与键的内积 = 相似度
softmax: 归一化成行随机矩阵 = 软选择
×V:  按权重凸组合所有值 = 软查询结果

本质: 把"哪个 token 该关注哪个"转成线性代数(内积+归一化)
```

**注意力是包装精美的矩阵乘法。** 线性代数给了它相似度、归一化、凸组合的全部能力。
