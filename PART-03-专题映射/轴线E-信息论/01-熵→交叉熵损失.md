# 熵 → 交叉熵损失

> 熵 $H$ 度量"一个分布有多不确定"，交叉熵 $H(p,q)$ 度量"用 $q$ 编码来自 $p$ 的数据平均要多少比特"。当 $p$ 是真实标签、$q$ 是模型预测时，最小化交叉熵就是逼模型预测逼近真实分布——这就是分类任务用交叉熵损失的全部数学。

**难度**：[标准]（需要概率分布 + 对数 + 一点编码直觉）

## 一、纯数学版：熵的三种等价定义

### 1.1 Shannon 熵（信息论视角）

分布 $p$ 的 Shannon 熵：

$$H(p) = -\sum_x p(x) \log p(x) = \mathbb{E}_{x \sim p}\!\left[\log \frac{1}{p(x)}\right]$$

**信息论解读**：$\log \frac{1}{p(x)}$ 是事件 $x$ 的**自信息**（小概率事件信息量大）。熵是自信息的期望——**平均不确定性**。

**例子**：
- 公平硬币 $p=(1/2, 1/2)$：$H = 1$ bit
- 偏置硬币 $p=(0.99, 0.01)$：$H \approx 0.08$ bit（几乎确定）
- 均匀分布 $p=(1/n, \ldots, 1/n)$：$H = \log n$（最大熵）

> 📖 概率与期望：[[Mathematics-Universe/05-概率论与数理统计/04-数字特征/数字特征详解.md]]

### 1.2 微分熵（连续情形）

对连续随机变量有密度 $p(x)$：

$$h(p) = -\int p(x)\log p(x)\,dx$$

**注意**：微分熵可为负（不像离散熵恒非负），且对坐标变换敏感。但**熵差**（如 KL）仍有意义。

### 1.3 极大熵原理

在已知约束（如均值、方差）下，**熵最大的分布**是最少先验假设的选择：

| 约束 | 极大熵分布 |
|------|-----------|
| 支集 $[a,b]$ | 均匀分布 |
| 均值 $\mu$，支集 $\mathbb{R}_+$ | 指数分布 |
| 均值 $\mu$，方差 $\sigma^2$ | **正态分布** |

**这解释了为什么正态分布 ubiquitous**——它是"只知道均值方差"时最诚实的假设。

> 📖 正态分布：[[Mathematics-Universe/05-概率论与数理统计/02-随机变量及其分布/随机变量及其分布详解.md]]

## 二、交叉熵与 KL 散度

### 2.1 交叉熵的定义

用分布 $q$ 编码来自 $p$ 的数据，平均需要的比特数：

$$H(p, q) = -\sum_x p(x) \log q(x) = \mathbb{E}_{x \sim p}\!\left[\log \frac{1}{q(x)}\right]$$

**关键不等式**：$H(p, q) \geq H(p)$，等号当 $p = q$。

### 2.2 KL 散度：交叉熵超出熵的部分

$$D_{\text{KL}}(p \| q) = H(p, q) - H(p) = \sum_x p(x) \log \frac{p(x)}{q(x)} \geq 0$$

**重要关系**：

$$\boxed{H(p, q) = H(p) + D_{\text{KL}}(p \| q)}$$

### 2.3 为什么分类用交叉熵

监督分类里：
- $p$ = 真实标签的 one-hot 分布（$p(y)=1$，其余 0）
- $q$ = 模型预测的 softmax 分布

训练目标：让 $q$ 接近 $p$。最小化 $D_{\text{KL}}(p \| q)$ 即可。但 $H(p) = 0$（one-hot 确定分布，熵为 0），所以：

$$D_{\text{KL}}(p \| q) = H(p, q) - \underbrace{H(p)}_{0} = H(p, q)$$

**最小化 KL = 最小化交叉熵**。这就是交叉熵损失的理论根据。

展开 one-hot 情形：

$$H(p, q) = -\sum_c p_c \log q_c = -\log q_y$$

其中 $y$ 是真实类。这就是**负对数似然（NLL）**。

## 三、从熵到 softmax：为什么这个输出层

### 3.1 softmax 的信息论起源

模型输出 logits $\mathbf{z}$，要转成概率。在"**所有概率分布中，使 $-\sum q_c \log q_c$ 受约束于期望 logits**"的极大熵问题下，解就是 softmax：

$$q_c = \frac{e^{z_c}}{\sum_k e^{z_k}}$$

**softmax 是 logit 约束下的极大熵分布**。

### 3.2 交叉熵 + softmax 的优美梯度

$$L = -\log q_y, \quad \frac{\partial L}{\partial z_c} = q_c - \mathbf{1}[c = y]$$

**梯度 = 预测误差**。错得越狠梯度越大——这就是分类任务学得快的原因。

## 四、工程实现层：PyTorch 交叉熵

### 4.1 多分类（最常用）

```python
import torch
import torch.nn.functional as F

logits = torch.randn(32, 10)              # batch=32, 10 类
labels = torch.randint(0, 10, (32,))      # 真实标签
loss = F.cross_entropy(logits, labels)    # 内含 softmax + NLL, 数值稳定
```

**永远传 logits，不要传 probs**——内部用 LogSumExp 技巧保证数值稳定。

### 4.2 手写交叉熵（理解内部）

```python
import torch

def cross_entropy(logits, labels):
    log_probs = F.log_softmax(logits, dim=1)        # log q
    nll = -log_probs[range(len(labels)), labels]    # -log q_y
    return nll.mean()

# 与 F.cross_entropy 数值一致
logits = torch.randn(4, 3)
labels = torch.tensor([0, 1, 2, 1])
print(cross_entropy(logits, labels), F.cross_entropy(logits, labels))
```

### 4.3 二分类（二元交叉熵）

```python
# 二分类: 用 sigmoid 输出 + BCE
logits = torch.randn(32)
targets = torch.randint(0, 2, (32,)).float()
loss = F.binary_cross_entropy_with_logits(logits, targets)
```

### 4.4 带 label smoothing 的交叉熵

```python
# 把硬标签 [0,0,1,0] 软化成 [0.025, 0.025, 0.925, 0.025]
# 缓解过自信, 提升校准
loss = F.cross_entropy(logits, labels, label_smoothing=0.1)
```

## 五、数值稳定性：LogSumExp

直接算 $\log\sum e^{z_k}$ 会溢出。等价变形：

$$\log\sum_k e^{z_k} = m + \log\sum_k e^{z_k - m}, \quad m = \max_k z_k$$

减去最大值后指数不会爆。PyTorch 的 `log_softmax` / `cross_entropy` 内部都用此技巧。

```python
# 错误写法(可能溢出)
probs = logits.exp() / logits.exp().sum()
log_probs = probs.log()

# 正确写法(数值稳定)
log_probs = F.log_softmax(logits, dim=1)
```

## 六、熵视角能解释的现象

### 6.1 类别不平衡：熵加权

少数类的熵大（不确定性高），多数类熵小。用 $1/H(p_y)$ 加权损失，能放大少数类贡献。

### 6.2 知识蒸馏：用软标签降熵

Teacher 模型输出软标签 $q^T$（非 one-hot，熵 $>0$），Student 学 $q^T$ 而非硬标签。**软标签的熵提供了"类间相似度"信息**（如"7 像 1"），Student 学到更丰富的结构。

### 6.3 信息瓶颈：用互信息压缩

中间表示 $\mathbf{z}$ 应**最大化与标签的互信息** $I(\mathbf{z}; y)$，同时**最小化与输入的互信息** $I(\mathbf{z}; x)$（丢掉无关细节）。这是交叉熵损失的信息论推广。

## 八、直觉总结

```
熵 H(p):     分布 p 有多不确定(公平硬币=1bit, 偏置硬币≈0)
交叉熵 H(p,q): 用 q 编码 p 的数据平均多少比特(≥H(p))
KL(p‖q):     q 比 p "多花"的比特(≥0, p=q 时为 0)

分类: p=one-hot(确定, H=0) ⇒ 最小化 KL = 最小化 CE = 最小化 NLL
      L = -log q_y, 梯度 = 预测误差, 学得最快
```

**交叉熵 = "你的预测分布离真实分布有多远"的比特度量。** 数学是信息论，工程是数值稳定的 softmax。
