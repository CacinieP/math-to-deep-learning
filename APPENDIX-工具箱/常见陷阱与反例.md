# 常见陷阱与反例

> 数学直觉在机器学习里会失效的场景。这些"反直觉"案例，每个都对应一个真实工程陷阱。

## 陷阱 1：梯度下降找的不一定是全局最小

**直觉陷阱**：认为训练就是"找最低点"。

**真相**：神经网络损失**非凸**，有大量局部极小、鞍点、平坦区。梯度下降只保证找到**局部**最小（甚至只是驻点）。

**反例**：$f(x) = x^4 - 3x^2 + 1$ 有两个局部最小（$x=\pm 1$）和一个局部最大（$x=0$）。从 $x=0.1$ 出发收敛到 $x=1$，从 $x=-0.1$ 收敛到 $x=-1$。

**工程对策**：多次随机初始化、动量、模拟退火。

## 陷阱 2：ReLU 网络的"死亡神经元"

**直觉陷阱**：ReLU 简单好用，越大越好。

**真相**：若某神经元长期输入负区，梯度恒为 0，**该神经元永久死亡**，不再更新。

```python
# ReLU' = 0 当 x < 0 ⇒ 梯度阻断
# 若权重初始化让某神经元总是输出负值 ⇒ 永远学不动
```

**工程对策**：Leaky ReLU、He 初始化、降低学习率。

> 📖 [[PART-04/02-梯度消失爆炸]]

---

## 陷阱 3：用 MSE 做分类

**直觉陷阱**：损失函数都差不多，随便选。

**真相**：MSE + sigmoid 在错得离谱处梯度**消失**（$\sigma'$ 饱和）。错得越狠学得越慢。

**数学**：$\frac{\partial}{\partial z}(\hat y - y)^2 = 2(\hat y-y)\sigma'(z)$，$\hat y \to 0/1$ 时 $\sigma' \to 0$。

**工程对策**：分类一律用交叉熵。

> 📖 [[PART-02/05-损失函数]]

---

## 陷阱 4：softmax 数值爆炸

**直觉陷阱**：$\frac{e^{z_i}}{\sum e^{z_k}}$ 数学上没问题。

**真相**：$z_i = 1000$ 时 $e^{1000}$ 远超 float32 上限（$\sim 10^{38}$），变 `inf`。

**工程对策**：减最大值 $m = \max z$，数学等价数值稳定：

$$\frac{e^{z_i - m}}{\sum e^{z_k - m}}$$

PyTorch 的 `F.cross_entropy` 内部已做。

> 📖 [[PART-04/01-数值稳定性]]

---

## 陷阱 5：忘记 train()/eval()

**直觉陷阱**：模型推理就是 `model(x)`。

**真相**：BatchNorm 在 train 模式用 batch 统计，eval 模式用移动平均。Dropout 在 train 启用 eval 关闭。**忘记切换会导致验证结果错误或不稳定**。

```python
model.train()  # 训练前
model.eval()   # 验证/推理前
```

> 📖 [[PART-04/03-BatchNorm]]

---

## 陷阱 6：过拟合 ≠ 训练损失还能降

**直觉陷阱**：训练损失下降说明在学好。

**真相**：训练损失降但验证损失升 = **过拟合**。模型在背训练集，没学到泛化。

**诊断**：画训练/验证损失曲线。两线分叉就是过拟合。

**工程对策**：早停、正则、数据增强、Dropout。

> 📖 [[PART-02/04-正则化]]

---

## 陷阱 7：维度不匹配的静默广播

**直觉陷阱**：PyTorch 会自动处理维度。

**真相**：广播规则可能**静默**产生错误结果。

```python
a = torch.randn(3, 4)
b = torch.randn(4,)
c = a + b        # (3,4) ✓ 正确广播
d = torch.randn(3,)
e = a + d        # (3,4)+(3,) ⇒ 广播失败或意外结果
```

**工程对策**：用 `assert x.shape == expected` 显式检查。

---

## 陷阱 8：梯度累加不清零

**直觉陷阱**：每个 batch 梯度自动重置。

**真相**：PyTorch 默认**累加**梯度。不清零会让梯度跨 batch 累积。

```python
opt.zero_grad()  # 必须每个 batch 调用
loss.backward()
opt.step()
```

> 📖 [[PART-02/02-反向传播]]

---

## 陷阱 9：小 batch 的 BatchNorm 失效

**直觉陷阱**：BN 稳定训练，越大越有用。

**真相**：batch 太小（如 2）时，batch 统计 $\mu, \sigma^2$ 估计极不准，BN 反而**伤害**训练。

**工程对策**：小 batch 用 GroupNorm/LayerNorm（不依赖 batch 维度）。

> 📖 [[PART-04/03-BatchNorm]]

---

## 陷阱 10：相关 ≠ 因果

**直觉陷阱**：模型学到的相关性 = 真实因果。

**真相**：神经网络是**函数逼近器**，只学相关性。虚假相关（如"晴天→野餐"因第三个变量"周末"）会被模型背下来。

**后果**：分布偏移时（训练 vs 真实世界）泛化崩塌。

**工程对策**：领域随机化、对抗训练、因果建模。

---

## 陷阱 11：评估数据泄漏

**直觉陷阱**：训练-验证-测试三划分够了。

**真相**：若同一对象的不同视图分散在训练集和验证集（如同一患者的多次就诊），验证集**不独立**，指标虚高。

**工程对策**：按**组**划分（group split），而非随机。

---

## 陷阱 12：deep 网络不一定好

**直觉陷阱**：越深越强。

**真相**：
- 太深 + 无残差 ⇒ 梯度消失，训不动
- 太深 + 过参数 + 少数据 ⇒ 严重过拟合

**工程对策**：用验证集选深度，配残差 + 正则。

> 📖 [[PART-04/02-梯度消失爆炸]]，[[轴线D/01-Weierstrass逼近]]

---

## 陷阱 13：学习率太大/太小的症状相似

**直觉陷阱**：损失不降就是学习率太小。

**真相**：学习率太大（发散震荡）和太小（龟速）都可能表现为"损失不降"。

**诊断**：
- 梯度范数大 + 损失震荡 ⇒ 学习率太大
- 梯度范数正常 + 损失平 ⇒ 学习率太小

**工程对策**：学习率范围扫描（lr finder）。

> 📖 [[PART-02/03-优化算法]]

---

## 陷阱 14：Universal Approximation ≠ 能学到

**直觉陷阱**：通用近似定理说神经网络万能。

**真相**：定理只说**存在**一组参数能逼近任意函数，**不保证梯度下降能找到**这组参数。所需宽度可能大到不可行。

**工程含义**：不能因为"理论上能"就忽视优化难度、数据量、正则。

> 📖 [[轴线D/01-Weierstrass逼近→通用近似定理]]

---

## 陷阱 15：迁移学习的负迁移

**直觉陷阱**：预训练总是有益的。

**真相**：源任务与目标任务差异大时，预训练权重可能**阻碍**学习（负迁移）。

**诊断**：从头训练 vs 微调，对比验证损失。

**工程对策**：选择性解冻层、降低预训练学习率。
